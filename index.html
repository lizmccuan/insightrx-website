Agent Architecture

Six Specialized Agents.
One Coordinated System.

Each agent has a defined role, a defined data source, and a defined output schema. They don't just run in sequence — they communicate. This is what makes InsightRx an intelligence system, not a pipeline of isolated AI calls.

📊
Analytics Agent
DATA LAYER · GA4 API
Ingests GA4 sessions, conversion events, traffic sources, and behavioral data. Runs anomaly detection algorithms to surface performance shifts, then encodes findings as structured signals for downstream agents to consume.
ga4_apianomaly_detectionconversion_analysis
🔍
SEO Agent
SEARCH INTELLIGENCE · GSC + SEMRUSH
Combines Google Search Console query data with SEMrush competitive intelligence to identify keyword ranking gaps, content cannibalization, technical health issues, and high-opportunity content angles.
gsc_apisemrush_apigap_analysistechnical_audit
✦
AEO / GEO Agent
AI SEARCH READINESS
Audits brand readiness for Answer Engine Optimization (AI assistant citations) and Generative Engine Optimization (AI-generated recommendations). Identifies entity clarity gaps, schema coverage, and content structure deficiencies invisible to traditional SEO tools.
entity_auditschema_analysisai_visibility
📱
Social Media Agent
SOCIAL INTELLIGENCE · META API
Pulls Meta Graph API data to analyze ad performance, audience engagement patterns, content decay rates, and creative performance scoring — feeding social signal context into the cross-channel strategy synthesis.
meta_graph_apiengagement_analysisroas_scoring
🧠
Strategy Agent
CROSS-CHANNEL SYNTHESIS
Receives structured JSON outputs from all four upstream agents via LangGraph handoffs. Identifies cross-channel correlations invisible from any single-platform view, scores opportunities by strategic priority, and prepares ranked input for the Recommendation Agent.
a2a_inputpriority_scoringpattern_recognition
🎯
Recommendation Agent
ACTION OUTPUT LAYER
Produces the final deliverable: a ranked list of strategic marketing actions, each with a rationale, expected impact score, effort estimate, responsible channel, and implementation notes. Output feeds directly into the Streamlit dashboard.
ranked_outputimpact_scoringstreamlit_render
🔄
Agent-to-Agent (A2A) Handoffs via LangGraph
InsightRx uses LangGraph to orchestrate structured communication between agents — not string concatenation or prompt chaining, but typed, schema-validated outputs that downstream agents consume as first-class input. This is the architectural decision that separates InsightRx from a set of sequential AI prompts: agents genuinely inform each other, creating emergent cross-channel intelligence that no single-agent system can produce.
Search Intelligence Layer

SEO. AEO. GEO.
Three Layers. One System.

Modern search visibility has three distinct layers. Most brands optimize only for one. InsightRx analyzes all three simultaneously — identifying exactly where a brand surfaces, where it disappears, and what to do about it.

🔍 SEO
Search Engine Optimization
The foundational layer — ranking in Google and Bing for high-intent keywords through technical health, content authority, and strategic link architecture. InsightRx's SEO Agent covers this comprehensively using live GSC and SEMrush data.
Real-time keyword ranking gap analysis vs. competitors
Technical health: Core Web Vitals, crawlability, schema coverage
Content opportunity scoring from live GSC impression data
Internal linking and on-page optimization recommendations
🤖 AEO
Answer Engine Optimization
The AI assistant layer — structuring content so ChatGPT, Perplexity, and Google AI Overviews extract and cite your brand as the authoritative answer. This requires a fundamentally different optimization approach than traditional SEO.
Identifying high-intent queries that AI assistants answer directly
Content structure for machine extractability: Q&A, schema, FAQs
Auditing whether brand content is cited in AI-generated answers
Topical authority clusters that LLMs recognize and trust
✦ GEO
Generative Engine Optimization
The frontier layer — ensuring brands surface when AI engines proactively recommend products and services in conversational responses, without a user clicking a link. This is where M2M brand evaluation begins.
Brand presence audit in generative AI responses across platforms
Machine-legible brand signals: entity definitions, structured data
Positioning content to influence AI retrieval and training patterns
Share-of-voice monitoring in zero-click AI recommendations

Why Grounding APIs
Change Everything
Traditional marketing AI tools analyze exports — last week's CSV, last month's report. Grounding APIs change the equation: InsightRx agents query live data sources directly, so every recommendation reflects current conditions, not historical snapshots.
⚡
Always-Current Intelligence
Because agents query live APIs, every insight reflects what's happening right now. A traffic anomaly that started yesterday surfaces in today's recommendation — not in next week's report review.
🔗
Cross-Source Correlation
When agents combine live GA4 data with live GSC rankings and live Meta performance, they identify correlations that static, siloed reporting can never surface. The pattern between a ranking drop and a traffic anomaly becomes visible in real time.
🔮
Foundation for M2M Intelligence
Grounding APIs are the infrastructure that will ultimately power machine-to-machine brand evaluation — where AI agents query brand data directly to make recommendations on behalf of users, without any human search behavior involved.
Forward Direction

Agent-to-Agent & M2M
Brand Decisions

The most significant shift in marketing isn't happening in ad platforms or SEO tools. It's happening in the infrastructure layer — where AI agents are beginning to evaluate, compare, and recommend brands without any human directing the search.

🤖
What A2A Means for Brands
When a user asks an AI assistant to recommend the best solution for a problem, that assistant doesn't search — it retrieves structured knowledge and evaluates options autonomously. Brands that aren't structured for machine-readability are invisible in this evaluation. InsightRx is designed to identify and close those gaps before competitors do.
🔄
Machine-to-Machine Evaluation
M2M brand evaluation describes the process by which one AI system queries another to validate or compare a brand — a scheduling agent checking vendor trustworthiness, a purchasing agent comparing supplier signals, a recommendation engine evaluating brand authority. The brand that wins is the one that speaks the machine's language: structured, authoritative, and entity-clear.
🏗️
What InsightRx Is Building Toward
InsightRx's AEO/GEO Agent already audits brand readiness for AI-mediated discovery. The roadmap extends this to active monitoring of brand citations across AI platforms, automated entity optimization, and GEO-specific content recommendations — preparing brands not just for today's search, but for tomorrow's agent-evaluated ecosystem.
📡
Zero-Click, AI-First Brand Presence
In AI-mediated discovery, recommendations arrive before a user sees a list of links. The brands that appear in those recommendations share one characteristic: their digital presence is structured for machine comprehension. InsightRx maps those structural requirements and builds optimization workflows around them — making brands legible to both humans and machines.
🔍
Machine-Legible Brand Signals
Structured data, entity clarity, and consistent brand signals that AI systems can parse and evaluate reliably when making recommendations.
🎯
Topical Authority Clusters
Deep content ecosystems establishing domain expertise — the authority signals LLMs prioritize when deciding which source to cite or recommend.
⚡
Proactive AI Positioning
Rather than reacting to algorithm shifts, InsightRx helps brands proactively build the technical and content infrastructure that positions them favorably in AI-mediated discovery — before competitors do.

Reflection

What Building InsightRx
Actually Taught Me

The technical work was the easy part. These were the real lessons.

🏗️
Systems Think Differently Than Campaigns
Designing InsightRx forced me to think in data contracts, schemas, and agent interfaces — not just marketing tactics. The shift from "what should I do?" to "how do I architect a system that answers that question automatically?" changed how I approach marketing strategy entirely. Every campaign now looks like a system waiting to be optimized.
🔗
Grounding Is the Hardest Part of AI
Getting agents to produce useful output is straightforward. Getting them grounded in real, live, clean data — handling API authentication, rate limits, data normalization, and schema mismatches — is where most AI marketing projects fail. InsightRx forced me to solve this at every layer, which gave me a practical understanding of production AI engineering that no course could provide.
🔮
AEO and GEO Are Already Real
I came into this project thinking AEO and GEO were theoretical. After auditing actual brands against AI assistant responses across ChatGPT, Perplexity, and Google AI Overviews, it became clear: brands are already winning and losing in these layers — they just don't know it yet. The window to build early advantage is narrow, and InsightRx is designed to identify it.
🎯
Strategy and Engineering Can't Be Separated
The best marketing strategy in the world is only as useful as the system that can execute and iterate it. And the most sophisticated AI pipeline is worthless if it isn't anchored in real strategic marketing questions. InsightRx forced me to hold both simultaneously — and that's the intersection I want to spend my career operating in.
